# 正式广告为什么是空白？

## 最常见：无填充（No Fill）

AdMob 返回 **错误码 1** 或文案含 `No ad to show` / `Request Error: No fill`，表示：

> 广告请求成功，但当前没有广告主愿意在你的地区/应用上展示。

在中国大陆 **非常常见**，尤其：

- 仅用 Google AdMob（个人开发者往往无法接穿山甲/优量汇）
- App 尚未在 App Store 上架
- 新创建的广告位（常需 **几小时～48 小时**）
- 使用 VPN/模拟器、流量过小的测试环境

**测试广告能显示、正式广告空白** → 几乎总是填充问题，不是代码写错。

## 请在 App 内查看具体错误

1. 设置 → 打开「使用正式广告位」
2. 看设置页 **广告调试 → 最近加载** 的错误信息
3. Xcode 控制台搜索 `[AdMob]`

## AdMob 后台自查

1. [AdMob](https://admob.google.com/) → 应用 → 确认 **Bundle ID** 为 `com.dizhenyujing.app`
2. 广告单元状态为 **有效**
3. 账户无政策限制、无付款/身份未验证提示
4. 若有 **企业/个体户** 资质，可在中介接穿山甲、优量汇（见 [MEDIATION_CN.md](./MEDIATION_CN.md)）

## 提升填充的建议（按优先级）

| 优先级 | 操作 |
|--------|------|
| 1 | **个人**：保持 AdMob，上架后有流量再观察；可试 AdMob 中介 **Unity Ads** |
| 2 | **有执照**：AdMob 中介接入穿山甲、优量汇 |
| 3 | App 上架后再观察（未上架填充率极低） |
| 4 | 真机 + 稳定网络测试 |
| 5 | 等待新广告位生效 24～48 小时 |

## 不要用正式广告自己狂点

调试正式位时请勿点击自己的广告，以免账号被封。

开发阶段请保持 **测试广告**（Debug 默认），上架后用 Release + 中介。
